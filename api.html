Character classes

Usage

VARCHAR("PATTERN", EXACT_AMOUNT_CHARACTERS)
VARCHAR("PATTERN", MIN_CHARACTERS, MAX_CHARACTERS)

Description

VARCHAR can be used in order to match patterns with varying characters. It can be used with either two or three arguments to alter its behavior where the first argument will be the characters in the pattern. Note that the order of the characters won't matter, since this will match any of them. This means "ACD135" and "CAD351" has the exact same meaning.

If you need to match all characters that ranges between two boundaries, you put a dash - between them, e.g. to match all uppercase characters between A and F, you provide "A-F" as the first argument. To match all digits between 0 and 9 would be "0-9". You can also combine multiple ranges in one VARCHAR, e.g "A-F0-9" could be used to match a hexadecimal number.

Matching exact amount of characters

By supplying only one boundary, VARCHAR will match the exact amount provided, e.g 7

VARCHAR("A-Z", 7)

Matching between a given range of characters

If you want to match patterns between a given boundary, e.g. 2 and 5, you can provide a number as the third argument to limit the upper boundary

VARCHAR("A-Z", 2, 5)

Matching characters with no upper limit

A lot of times you have to match patterns where you don't know the upper limit, then you can provide an asterix * as the second argument, e.g. if you want to match all patterns with "A-Z" that contains of "two or more" characters, you do this by writing

VARCHAR("A-Z", 2, *)

Matching Literal Text

Usage

@"literal text to match"

Description

In a lot of cases when working with regular expressions, you will have to match literal words or sequences of text that doesn't change in the pattern. The way we do that is by putting it in a string beginning with an @, which means that you want to match the EXACT word/text provided. Although the intended use are for matching literal text, it can be used to inline standard regular expressions if required.

Example usage

Here we want to match the word "sample_text", which is simply done by writing

@"sample_text"

If you want to escape characters, you do that by putting a backslash \ before the character you want to escape, e.g if you need to match quotes in the literal text, you would write
@"\"quoted string\""


ANY (OR operation)

Usage

ANY(IDENTIFIER_1, ...)

Description

The API call "ANY" is basically an OR operation which takes variadic arguments, e.g you can supply any amount of arguments. It requires at least one argument.
